    Overview

    Abstract

    This documentation defines the specifications of an API who’s responsibility is to provide access to Pruvo’s unique price monitoring capabilities.

    The API is designed as stateless and RESTful and responds only with JSON format. For security considerations, we require the use of SSL(HTTPS) only.

    Current API version: 1.0

    Request data

    • Every string passed to/from API has to be UTF-8 encoded
    • Support for SNI SSL is required
    Every partner is subjected to a QPM(Queries Per Minute) limitation. Current QPM limit is 50.

    Versioning

    We may update the API without increasing the version number (i.e 2.1 2.2 etc…) for NON-BREAKING changes.

    Authorization

    This section describes the authorization process.

    Every request to any available endpoint requires authorization. To get the authorization credentials you need an active partner account registered within Pruvo’s systems.

    The following table details the required headers that must be sent with each request.

    Header key Header value Description
    Authorization Authorization Key Key provided by Pruvo
    API-Version Api Version PI Version to use

    If you prefer not to send the authorization data as headers, you can send them as query parameters:

    Key Value Description
    access_token Authorization Key Key provided by Pruvo
    api_version Api Version PI Version to use

    Meal Plans

    0 no meal plan
    1 breakfast
    2 half board
    3 all inclusive
    4 full board

    Push end-point provides you with the ability to submit a booking for price monitoring.
    URL /yourEndpoint
    SCHEMA HTTPS
    METHOD POST

    Accepted parameters

    Name Format Description Mandatory Comments
    uniqueId String an identifier for correlating with your platform True
    email Email String account to import the reservation to True
    currencyCode ISO-4217 String booking currency True
    price Number total price (excluding taxes paid at the hotel) True
    tax Number taxes included in the price (excluding taxes paid at the hotel) Optional
    roomType String name of the room True
    rooms Number the number of rooms booked True
    persons Number the total number of persons in all the rooms True
    children Number the total number of children in all rooms True
    arrivalDate MM/DD/YYYY arrival date to the hotel True must be a future date
    departureDate MM/DD/YYYY departure date from the hotel True must be after arrivalDate
    lastFreeCancelTime MM/DD/YYYY the last date to cancel the booking for free. True must be before arrivalDate. Omit if not a free cancellation reservation
    mealPlan Number (Enum) included meal plan True
    hotelName String hotel name True
    address String hotel street address True
    city String hotel city True
    country String hotel country True
    latLong Geo String (lat,long) geo location of the hotel True example: 32.78778,-79.93472
    manageLink Url String link to refer user to modify/cancel booking True
    thumbnail Url String url to hotel picture Optional
    hotelLink Url String link to hotel page in the site Optional
    hotelCode String the ean hotel code Optional
    tripId String the tripId provided to the user for this booking (used only when required by pruvo) Optional
    custom String custom data (used only when required by Pruvo) Optional
    nationality ISO 3166-1 Alpha-2 String booker nationality Optional Recommended for better results
    A single request represent a single booking information. It can contain multiple rooms, only if they are exactly identical
    price and tax parameters should be given in the `currencyCode` currency value.
    (`persons` + `children`) / `rooms` must be a positive integer

    Error Handling

    Error Response Structure

    All 400 errors encountered during request will render the same error structure.

    400 Error Response Structure

    
{
    "error": {
        "name": "Error",
        "status": 400,
        "message": "Request data validation error",
        "errors": [
            {
                "failed": "required", // The condition that failed
                "path": [
                    "hotelName" // parameter that caused an error
                ]
            }
        ]
    }
}

    The API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- The request cannot be fulfilled due to bad syntax.
    401 Unauthorized -- The end-point requested is not allowed.
    403 Forbidden -- The end-point requested is not allowed.
    404 Not Found -- The end-point requested could not be found.
    405 Method Not Allowed -- You tried to access an end-point with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The request end-point requested has been removed from our servers.
    429 Too Many Requests -- You're making too many requests! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
